## 数据构建方法

### 第一阶段：分类体系建立

1. 用 Claude Opus 生成跨领域数学结构分类
2. 用 DeepSeek R1 独立生成分类
3. 合并去重，得到84种结构类型

### 第二阶段：跨域描述生成

对每种结构类型，生成多个不同领域的自然语言描述：

```json
{
  "type_id": "07",
  "type_name": "逻辑斯蒂增长",
  "domain": "流行病学",
  "description": "传染病在人群中的传播初期呈指数增长，但随着易感人群减少，传播速率放缓并最终饱和，累计感染人数呈S形曲线"
}
```

每条描述需满足：
- 不提及结构类型名称（避免模型学到shortcut）
- 使用目标领域的专业术语
- 描述具体现象，而非抽象数学

### 第三阶段：质量审计

进行了4轮审计（audit-1 到 audit-4），每轮检查：
- 描述是否准确反映了对应的结构类型
- 是否有领域错误
- 是否泄露了结构类型信息
- 跨类型的描述是否有混淆

审计后清洗数据，最终得到 `clean.jsonl`（1214条）。

## 训练对的构建

使用对比学习（Contrastive Learning），需要正负例对：

**正例对**：同一结构类型的不同领域描述
- 例如："细菌在培养基中的增长先快后慢最终趋于平衡" 和 "新产品在市场上的渗透率呈S形增长"
- 两者都是逻辑斯蒂增长（类型07），但分属生物学和市场营销

**负例对**：使用 MultipleNegativesRankingLoss 自动构建
- 同一 batch 中的其他样本自动成为负例
- 不需要手动标注负例
