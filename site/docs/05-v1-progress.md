# V1：结构同构搜索引擎

> V1 的目标：输入一个科学概念的描述，返回在不同领域中具有相似底层结构的概念。

## 分类体系

### 设计思路

要让模型学会"结构同构"，首先需要一个明确的结构分类体系。我们的做法是：用数学结构作为桥梁——不同领域的现象如果背后对应同一种数学结构，就存在深层同构。

### 20 大类、84 种数学结构

| 大类 | 包含结构数 | 典型结构 |
|------|----------|---------|
| 指数与对数 | 5 | 指数增长/衰减、对数标度、幂律 |
| 线性与仿射 | 4 | 线性映射、仿射变换、叠加原理 |
| 周期与振荡 | 5 | 简谐振荡、参数共振、自激振荡 |

### 分类原则

1. **颗粒度适中**：太粗（如"线性"）区分力不够，太细（如"三阶常微分方程"）覆盖面太窄
2. **跨域适用**：每种结构至少能在 3 个不同领域中找到实例
3. **可操作**：给一段自然语言描述，应该能明确判断属于哪种结构
4. **互斥性**：减少一个现象同时属于多种结构的情况（虽然完全互斥不现实）

## 训练数据

### 数据生成流程

```
84 种结构 × 15 条/结构 = 1260 条（目标）
    ↓
GPT-4 + Claude 生成（每种结构指定不同领域）
    ↓
质量审核（人工 + LLM 双审）
    ↓
放宽标准（允许"跨领域但不够远"的样本）
    ↓
1214 条干净数据
```

### 数据格式

每条数据为 JSONL 格式：

```json
{
  "type_id": 3,
  "type_name": "exponential_decay",
  "domain": "pharmacology",
  "description": "药物进入人体后，血药浓度随时间按固定半衰期下降。每经过一个半衰期，剩余药物浓度减半。这个过程可以用指数衰减函数 C(t) = C₀ × e^(-kt) 描述，其中 k 是消除速率常数。"
}
```

### 字段说明

| 字段 | 类型 | 说明 |
|------|------|------|
| type_id | int | 数学结构编号（1-84） |
| type_name | string | 数学结构英文名 |
| domain | string | 所属领域（英文） |
| description | string | 该领域中这种结构的自然语言描述（中文） |

### 数据质量控制

**生成阶段**：
- 每种结构指定 15 个不同领域，确保多样性
- prompt 要求描述具体、避免直接提及数学公式名称
- 同一结构的不同描述之间语义距离要足够远

**审核阶段**：
- 自动审核：检查描述长度（>50 字）、是否提及结构名称（不应该）、领域多样性
- 人工审核：抽样 20% 检查准确性
- LLM 审核：让另一个 LLM 判断描述是否准确反映了对应的数学结构

**放宽标准**：
初始审核过于严格（要求领域间"完全不同"），导致通过率只有 80%。放宽到"至少不是同一学科的子方向"后，通过率提升到 96%，最终得到 1214 条。

### 数据分布

- **每种结构的样本数**：平均 14.5 条（min 12, max 15）
- **领域覆盖**：物理、化学、生物、经济学、心理学、计算机科学、社会学、工程学、医学、生态学等 30+ 领域
- **描述长度**：平均 120 字（min 55, max 280）

## 模型

### 基座模型

**text2vec-base-chinese**（shibing624/text2vec-base-chinese）

- 架构：BERT-base，12 层 Transformer
- 参数量：~110M
- 维度：768 维
- 预训练：中文语料上的对比学习预训练
- 选择原因：中文表现好、体积适中（不需要大模型）、社区生态完善

### 微调方法

**对比学习（Contrastive Learning）**

损失函数：MultipleNegativesRankingLoss（MNRL）

```python
from sentence_transformers import SentenceTransformer, losses

model = SentenceTransformer('shibing624/text2vec-base-chinese')
train_loss = losses.MultipleNegativesRankingLoss(model)
```

**训练逻辑**：
- 同一 type_id 的不同描述构成正样本对
- 不同 type_id 的描述构成负样本对（in-batch negatives）
- 优化目标：让同一数学结构的跨域描述在向量空间中靠近，不同结构的描述远离

**超参数**：
- batch size: 64
- learning rate: 2e-5
- epochs: 10
- warmup ratio: 0.1
- 评估策略：每 epoch 结束后在验证集上评估

### 评估指标

训练完成后，用以下指标评估模型质量：

| 指标 | 说明 | 目标 |
|------|------|------|
| Silhouette Score | 聚类质量，-1 到 1 | > 0.3 |
| Retrieval@5 | top-5 检索命中率 | > 0.6 |
| 基线对比 | vs. 未微调的 text2vec | 显著提升 |
| 跨域一致性 | 同结构不同域描述的平均相似度 | > 0.7 |

## 推理流程

训练完成后，V1 的使用流程：

```
用户输入：一段科学现象描述
    ↓
模型编码：转化为 768 维向量
    ↓
向量检索：在预计算的知识库中找最近邻
    ↓
结果过滤：排除同领域结果、排除低分结果
    ↓
输出：top-K 个跨域结构同构候选
```

### 示例

**输入**："在生态系统中，当捕食者数量增加时，猎物数量减少；猎物减少导致捕食者食物不足而减少；捕食者减少后猎物数量又恢复。这种此消彼长的关系导致两个种群的数量呈现周期性波动。"

**期望输出**：
1. 宏观经济中的存货周期（经济学，结构：负反馈振荡）
2. 化学振荡反应中的 Belousov-Zhabotinsky 反应（化学，结构：极限环）
3. 心脏起搏细胞的电位振荡（医学，结构：自激振荡）

这些候选来自完全不同的领域，但底层数学结构（周期性振荡由负反馈延迟驱动）是相同的。

## 已知局限

1. **数据量偏小**：1214 条对于 84 类的分类任务来说不算多，可能导致长尾类别的表现差
2. **中文偏向**：模型和数据都是中文的，跨语言能力有限
3. **静态知识库**：V1 阶段不能处理知识库外的新概念
4. **粒度固定**：84 种结构的粒度可能对某些任务太粗或太细

这些局限将在 V2 和后续版本中逐步解决。
