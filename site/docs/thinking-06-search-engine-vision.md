# 结构同构搜索引擎的设想

> 思考链路第6节 | 2026-04-11 深夜讨论

## 核心问题

理论框架已经建立（创新≈结构迁移），shadow mode 提供了运行机制。现在的问题是：**怎么把它变成一个可运行的系统？**

## 讨论过程

### 核心设计思路

用户输入一个问题（如"如何解决城市交通拥堵"），系统做以下事情：

1. **结构提取**：分析问题的底层数学结构（交通拥堵 ≈ 网络级联/渗流阈值）
2. **跨域搜索**：在知识库中搜索共享相同结构的其他领域现象
3. **排序输出**：按语义距离×结构匹配度排序，输出最远但最匹配的结果

例如输出：
- 排名1：电力网络的级联故障防御（网络科学，相似度0.92）
- 排名2：传染病的隔离策略（流行病学，相似度0.88）
- 排名3：河流防洪的分流设计（水利工程，相似度0.85）

### 技术架构

```
输入层：
  自然语言问题 → LLM提取结构描述 → 结构embedding

匹配层：
  结构embedding → 在知识库中做向量相似搜索
  知识库 = 大量科学/工程/社会现象的结构化描述

过滤层：
  移除同领域结果（太近，没价值）
  移除已知类比（已被发现的跨域连接）
  移除表面相似但结构不同的配对

输出层：
  排序后的跨域结构匹配列表
  每个结果附带：迁移路径说明、可操作建议
```

### 关键技术挑战

**挑战1：结构提取的准确性**

怎么把自然语言描述的现象准确映射到数学结构？这是最核心的问题。

方案：用 embedding 模型做，而不是规则匹配。训练一个模型，让它把"描述不同但结构相同"的文本映射到相近的向量空间位置。

这就是V1要做的事：fine-tune 一个 embedding 模型，使其能识别跨域的结构同构。

**挑战2：知识库的覆盖度**

需要一个足够大、足够多样的"已知现象"库。每个现象需要：
- 自然语言描述
- 所属领域标签
- 结构类型标签
- 数学表达（如果有的话）

这就是V2的知识库：500个跨学科现象。

**挑战3：区分"结构同构"和"表面类似"**

"股市涨跌像过山车" 是隐喻，不是结构同构。
"股价随机游走和布朗运动共享同一个随机微分方程" 是结构同构。

模型需要学会区分这两者。训练数据中需要包含正例（真正的结构同构对）和负例（表面类似但结构不同的对）。

### 为什么选择 embedding 模型

1. **可扩展**：新现象只需要编码一次，搜索是O(N)或O(logN)
2. **容错**：即使描述不精确，向量空间中的近似搜索也能工作
3. **可解释**：可以分析两个向量为什么接近（哪些维度贡献了相似度）
4. **可迭代**：模型精度不够就加数据重新训练

### 分阶段实施计划

| 阶段 | 目标 | 方法 |
|------|------|------|
| V1 | 训练结构同构embedding模型 | 84种结构×1214条数据，对比学习 |
| V2 | 在知识库上做全面扫描 | 500个现象两两比对，过滤已知，输出候选 |
| V3 | 开放API | 用户输入问题，实时返回跨域匹配 |
| V4 | 自主发现 | Shadow mode持续运行，自动生成候选假说 |

## 得出的结论

1. 系统架构清晰：embedding模型 + 知识库 + 过滤管道
2. V1是基础（模型质量决定一切），V2是第一个有价值的产出
3. 关键风险：embedding模型能否真正学会区分结构同构和表面类似
4. 选择 text2vec-base-chinese 作为基座，用对比学习fine-tune

## 关键引用

> "这个系统的核心不是'搜索'，而是'embedding'——如果模型能在向量空间中正确地把结构相同的东西放在一起，搜索就自然解决了。"

> "V1是地基。如果embedding模型不行，后面的一切都是空中楼阁。"
