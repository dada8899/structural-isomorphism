# 路线图

## 总体愿景

从一个简单的搜索引擎，逐步演进为一个能辅助科学假说生成的 AI 系统。每个阶段都是独立可用的产品，同时为下一阶段打基础。

```
V1 搜索引擎 → V2 异常探测器 → V3 影子模式 → V4 科学发现系统
  (人问系统答)   (系统主动扫描)   (从偏差中发现)   (多领域协同)
```

## V1：结构同构搜索引擎（当前阶段）

### 时间线：2026 年 4 月

| 里程碑 | 状态 | 说明 |
|--------|------|------|
| 分类体系设计 | 已完成 | 84 种数学结构，20 大类 |
| 训练数据生成 | 已完成 | 1214 条高质量标注数据 |
| 模型训练 | **进行中** | text2vec 对比学习微调 |
| 模型评估 | 待开始 | Silhouette + Retrieval@5 |
| CLI 工具 | 待开始 | 命令行查询接口 |
| 效果验证 | 待开始 | 10 个测试查询的定性评估 |

### 交付物

1. 训练好的向量模型（可下载的 checkpoint）
2. 命令行查询工具
3. 评估报告（包含所有指标）
4. 训练数据和分类体系（可复现）

### 成功标准

- Silhouette Score > 0.3
- Retrieval@5 > 0.6
- 定性评估：10 个测试查询中至少 6 个返回有意义的跨域结果

## V2：异常探测器（1-2 个月）

### 时间线：2026 年 5-6 月

| 里程碑 | 预计时间 | 说明 |

| 管道运行 | 第 3 周 | 自动扫描 124,750 对 |

### 交付物

1. 500 个科学现象知识库

3. 发现报告：30-50 个候选跨域同构
4. （可选）arXiv 预印本

### 成功标准

- 至少发现 10 个之前未知的有价值跨域同构
- 假阳性率 < 50%（最终候选中至少一半经得起人工审核）
- 管道端到端运行时间 < 24 小时

## V3：影子模式原型（3-6 个月）

### 时间线：2026 年 7-10 月

### 核心思路

选择一个具体领域（初步计划：金融市场），部署影子模式：
1. 建立领域预测模型
2. 持续运行，记录预测偏差
3. 用 V1/V2 的方法分析偏差模式
4. 寻找"这个偏差模式在其他领域有已知解释"的情况

### 为什么选金融

- **数据丰富**：公开行情数据无限量
- **反馈快速**：预测结果每天可以验证
- **实用价值**：如果能发现有价值的跨域洞察，直接有商业应用
- **容错性好**：预测错了也不会有物理伤害

### 关键技术挑战

1. **偏差分类**：不是所有预测失败都有意义，需要区分"噪声导致的偏差"和"系统性的偏差模式"
2. **实时性**：影子模式需要持续运行，对系统稳定性有要求
3. **跨域匹配**：从金融偏差模式到其他领域的结构匹配，需要更强的模型能力

### 交付物

1. 金融领域影子模式原型（可运行的系统）
2. 偏差分析报告（至少运行 1 个月的数据）
3. 跨域洞察候选列表
4. 技术文档

## V4：多领域科学发现系统（远期愿景）

### 时间线：2027 年

### 愿景

将 V1-V3 的能力扩展到多个科学领域，形成一个通用的科学发现辅助系统：

- **多领域知识库**：从 500 扩展到 5000+ 科学现象
- **多领域影子模式**：同时在多个领域运行预测和偏差检测
- **自动假说生成**：基于发现的同构和偏差，自动生成可检验的科学假说
- **研究者协作平台**：让不同领域的研究者能够看到系统发现的跨域关联

### 实现前提

V4 的实现取决于 V1-V3 的结果：
- V1 验证了"结构同构可以被模型学习"
- V2 验证了"大规模扫描能发现有价值的同构"
- V3 验证了"影子模式能在真实场景中工作"

如果任何一个阶段的结果不够好，都需要调整后续路线。

## 风险与应变

### 技术风险

| 风险 | 概率 | 影响 | 应变 |
|------|------|------|------|
| V1 模型效果差 | 中 | 高 | 换更大的基座模型 / 增加数据量 |
| V2 发现太少 | 中 | 中 | 扩大知识库 / 降低阈值 |
| V3 偏差太噪声 | 高 | 中 | 先用模拟数据验证方法 |
| 论文被拒 | 中 | 低 | 换期刊 / 补充实验 |

### 资源风险

- **个人项目**：没有团队，进度完全取决于个人时间投入
- **计算资源**：V1 本地 GPU 够用，V3 可能需要云资源
- **API 成本**：V2 预估 $30，可控

### 关键假设

整个路线图基于以下假设，如果任何假设被证伪，需要调整方向：

1. ~~创新 = 跨域迁移~~ → 已修正为"~60% 的创新涉及跨域迁移"
2. 数学结构是有效的中间表示
3. 对比学习可以让模型忽略领域差异、聚焦结构相似性
4. 500 个现象的知识库足够大，能产生有意义的发现
