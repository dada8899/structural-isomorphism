ABOUT PHASE DETECTOR

一个关于「用数学结构看公司」的实验

⚠️ 先说清楚：这是一个实验性产品
我一个人（个人项目，非商业实体）做的。数据由 LLM 生成，可能不准；覆盖面有限（目前 204 家公司）；不构成任何投资建议。你看到的每一份分析都应该作为「多一个思考视角」，不是决策依据。如果你看到 Phase Detector 说某家公司像 1998 年的 Cisco，请自己去验证这个类比成不成立，不要直接拿来做投资。

Phase Detector 在问一个简单的问题：如果我们不看市盈率、不看股息率、不看分析师评级，而是用物理学、生物学、生态学里研究了一百年的经典动力学方程来描述公司，会看到什么？

一、为什么做这个

这是 Structural 学术版 的延伸。学术版在做一件事：在不同领域（物理、生物、经济、社会）里找「数学结构相同」的现象——比如传染病传播、技术扩散、金融泡沫、社会运动，表面完全不同，但底层方程是同一个 SIR 模型的变体。

商业版把这个思路落在单一领域：公司。如果不同学科的现象在数学结构上是同构的，那同一套方程也应该能描述公司的动力学。

Netflix 的订户增长和兔子种群生长有什么关系？—— 数学上是同一个方程（Verhulst 逻辑斯蒂模型）。

二、核心原理（大白话）

第一步：把每家公司分类到一个「动力学家族」

用 LLM + 结构化 prompt，给 204 家上市公司打标签，从 24 种经典动力学方程里选一个最匹配的：

  • ODE1_logistic（S 曲线饱和）—— Netflix / Shopify / 拼多多这种"从爆发到饱和"的公司
  • ODE1_exponential_growth（指数增长飞轮）—— NVIDIA / Broadcom 这种当前正反馈飞轮未见顶的
  • Bistable_switch（双稳态切换）—— Tesla 这种在 niche 和主流之间翻转的
  • Fold_bifurcation（鞍结跳变）—— Peloton 这种经历不可逆崩塌的
  • Phase_transition_2nd（二阶相变，临界放缓）—— Meta / Snap 这种接近拐点的
  • ... 还有 19 个家族

第二步：判断公司当前处于什么「相位状态」

7 个 canonical phase：growth_phase / stable / saturated / approaching_critical / post_transition / unstable / contracting。每种对应不同的投资逻辑。

第三步：找出可观测的「临界点」

每家公司配 2-5 个 critical points——不是"市场情绪变化"这种虚的，而是"月活跌破 X 亿"、"ARPU 增速首次超过订户增速"这种具体可查的。

第四步：把"像不像"变成可检索

有了结构标签之后，"Tesla 现在像 1998 年的 Apple"不再是一句文艺话，而是一个可以通过结构匹配回答的具体问题。类比引擎就是干这个的。

三、已经验证过的事（包括失败）

  • ✓
    LLM 能做出一致的结构分类：204 家公司的 dynamics_family 标签，用同样的 prompt 多跑几次，结果稳定。分布也合理（不会 80% 都是 ODE1_saturating 这种退化情况）。
  • ✓
    10 份样例深度报告可用：用 Verhulst / SIR / Gause / Ising 等经典模型分析 Netflix / NVIDIA / Tesla / 拼多多等公司，每份 1500-2500 字。数据真实、论证成立。
  • ✓
    类比引擎给出有历史依据的类比：Netflix → Comcast 2002 / Sirius 2007 / Tencent Music 2018；Tesla → Apple 2006 / BlackBerry 2009 / Boeing 2011。类比基于真实数据，不是 LLM 瞎编。
  • ⚠
    相位演变时间线仅覆盖 20 家：NFLX / NVDA / TSLA / PTON / BYD / SHOP / 美团 / SNOW / 理想 / Apple / Amazon / Meta / Google / Microsoft / Intel / 茅台 / 腾讯 / ASML / BABA / PDD。每家 3-4 个历史节点。LLM 生成的年份和数字可能有偏差，需自行校对。
  • ✗
    PELT 结构拐点检测不 work：我做过一次 POC，对 25 家公司跑 PELT 算法自动找历史结构断点。54% 对应真实 regime 变化，但去掉宏观事件后真信号只剩 12%——说明"每周扫全市场找结构拐点"这条路 ROI 太低。所以最终选择了「类比引擎 + 时间线」的方向：不预测，只展示。
  • ✗
    量化信号回测没做：没有"用 Phase Detector 的分类选股，夏普比多少"这种证明。目前的价值是「分析视角」，不是「量化工具」。
  • ✗
    没有真实用户反馈：0 用户，所有改进都基于我自己的判断。这是目前最大的风险——可能整个方向是自嗨。

四、未来可能的场景（没承诺会做）

如果有人真的用起来了（或者我自己继续折腾），下面这些是可能的方向：

  1. 组合画像：上传持仓 → 看组合的结构分布。"你 80% 在 positive_loop 公司，0% 在防御型"——机构会愿意付钱的视角。
  2. 公司对比：选两家直接对比它们的 structural 异同、共同类比、对立预测。
  3. 周度 digest 邮件：每周给订阅者一份"本周结构有变化的公司 + 历史类比 + 启示"。
  4. 扩到 500+ 家：现在 204 家覆盖有限。如果信号有效，扩面值得做。
  5. 分享卡 + 永久 URL：每份类比分析能生成漂亮的分享卡，发到 Twitter / 小红书有传播效果。
  6. 时间维度深化：每家公司不只看"现在"，看它 15 年走过哪几个阶段、每次切换的触发条件——相位演变时间线已经是第一步，未来可扩到 50-100 家。

这些都可能做，也可能都不做。取决于用户反馈和我个人的时间/兴趣。

五、技术上用了什么

  • 数据层：204 家公司的 struct tuples（手工定义 schema + LLM 批量生成 + 人工校对）
  • 分析层：OpenRouter 接 Claude Sonnet 4.5（深度分析）+ Kimi K2 / Grok-4-fast（快速检索）
  • 前端：纯静态 HTML + vanilla JS + KaTeX（公式）+ marked.js（Markdown）
  • 后端：FastAPI + httpx（async streaming）
  • 所有内容：开源（GitHub）
