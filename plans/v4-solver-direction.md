## 背景

V1（embedding 模型）+ V2（候选对生成管道）已验证用户核心框架的前两步："高语义距离 × 高结构同构"。但第三步——**关键变形**——完全空白。

同时，V1+V2 的使用方向是被动的：遍历知识库吐出发现给用户消费。V3 把方向翻过来：用户给一个 B 领域待解问题，系统输出 top-3 跨域解法 + 变形说明。

两件事叠加，构成论文叙事的闭环。

---

## 架构

```
B 领域问题（自然语言）
  ↓
[1] 问题结构化器          （复用 V1 + 84 类分类器 + NL→签名 prompt）
  ↓
B 结构签名
  ↓
[2] 反向同构检索          （V2 翻转方向，top-k 精度优先）
  ↓
top-k A 候选（已解问题）
  ↓
[3] 变形预测器 ★ V3 核心
  ↓
变形序列
  ↓
[4] 解法生成器 + critic   （Opus 生成 / GPT-5 独立验证）
  ↓
top-3 B 解法 + 置信度 + 变形解释
```

## 模块 3：变形预测器（V3 核心创新）

### 任务定义

**输入**：
- 现象 A 的结构签名 + 已知解 X
- 现象 B 的结构签名 + 待解问题
- A↔B 映射关系

**输出**：
- 从 X 迁移到 B 领域所需的变形操作序列
- 每个操作含：类型 + 参数 + 理由

### 7 种变形基元

| 类型 | 描述 | 示例 |
|------|------|------|
| 量纲替换 | 把某个维度换成异域概念 | F=ma → 信息扩散中 m=信息惯性 |
| 连续/离散切换 | 连续方程 ↔ 差分方程 | 微分方程 → 元胞自动机 |
| 因果方向翻转 | 因果箭头反向 | 供需决定价格 → 价格决定供需 |
| 边界条件改写 | 开放↔封闭、初始↔稳态 | 开放系统 → 封闭系统版本 |
| 维度升降 | 降维或升维 | 2D 格点 → 3D |
| 随机性注入/移除 | 确定性↔随机 | 确定性 ODE → 蒙特卡洛 |
| 时间尺度缩放 | 慢过程↔快过程 | 热力学平衡 → 非平衡动力学 |

**覆盖率验证**：7 种是否够用，用验证管道产出的 tier-1 幸存发现做回归测试——看每个发现的"变形路径"能否被这 7 种覆盖。不够就扩基元集。

### 训练数据构建

**目标规模**：200-500 条科学史跨域迁移案例

**每条标注**：
```json
{
  "source_domain": "经典力学",
  "source_concept": "简谐振子",
  "source_equation": "m·ẍ + k·x = 0",
  "target_domain": "电路理论",
  "target_concept": "LC 振荡回路",
  "target_equation": "L·Q̈ + (1/C)·Q = 0",
  "transformations": [
    {"type": "量纲替换", "from": "m→L, k→1/C", "reason": "..."},
    {"type": "量纲替换", "from": "x→Q", "reason": "..."}
  ],
  "validated": true,
  "source_refs": ["..."]
}
```

**构建流程**：
1. **用户种子**（20-30 条）：用户提供经典案例——麦克斯韦妖 ↔ 信息熵、Lotka-Volterra ↔ 经济周期、热传导 ↔ 扩散-期权定价 等
2. **LLM 扩展**（到 200 条）：Opus + 多样性采样，覆盖物理/生物/经济/信息论/社会学五大域
3. **用户抽审**：随机抽 50 条人工纠错，修正变形类型标注和方程
4. **重新扩展**：用纠错结果反哺 LLM 重新生成剩余 150 条
5. **补充现代案例**：从验证管道 Layer 2+3 幸存的"真新"发现里抽 20-50 条补进数据集，对冲科学史老案例的分布偏差

### 模型选型

**Option A（先做）**：V1 embedding + 小分类器头
- 架构：V1 embedding（冻结）→ 两层 MLP → 多标签分类（变形类型 + 序列位置）
- 优点：训练快、成本低、可解释
- 缺点：变形参数（具体怎么替换）需要后接一个 LLM 填充

**Option B（兜底）**：Opus / Kimi fine-tune
- 架构：直接 fine-tune 成"变形专家"，端到端输出变形序列 + 参数
- 优点：质量可能更高
- 缺点：训练贵、调试难、黑盒

**先跑 Option A 看离线指标**，不够再上 B。

## 模块 1：问题结构化器

**输入**：自然语言问题（如"为什么创业公司早期更容易做出突破性创新？"）
**输出**：结构签名（系统类型 + 约束 + 优化目标 + 反馈环等）

**实现**：
- 复用 V1 embedding + 84 类结构分类器
- 新增一层 NL→签名 prompt engineering（Opus + few-shot）
- 关键挑战：自然语言问题往往模糊，需要"澄清对话"环节——让 LLM 先追问 2-3 个细化问题再定签名

---

## 模块 2：反向同构检索

V2 管道是"遍历全库 → 全量候选对"，V3 翻转为"给定 B 结构 → A 库里找 top-k"。

**关键差异**：
- V2：覆盖率优先（找到尽可能多的发现）
- V3：精度优先（top-3 必须对）

**Ranker 重训**：
- 不只用 embedding 相似度
- 加新信号：A 领域是否有"可迁移的已知解"（需要构建 A 库时标注每条是否有解）
- 加对立采样：把"表面相似但不同解"的对做负样本

---

## 模块 4：解法生成器 + Critic

**Generator（Opus）**：
```
输入：B 问题 + A 候选 + A 已知解 + 变形序列预测
输出：B 问题的候选解法（自然语言 + 关键公式）
```

**Critic（GPT-5 或 Gemini-2.5-Pro，独立模型）**：
```
输入：B 问题 + Generator 输出的解法
不看：A 候选、变形序列（避免被说服）
输出：解法是否真正解决 B 问题，0-1 打分 + 理由
```

**关键设计**：Generator 和 Critic 必须用不同模型，避免同上下文自我认同。

## 评估

### 离线

- **变形序列预测准确率**：在历史案例测试集上，预测变形序列与标注的序列相似度（用编辑距离）
- **top-k 召回率**：给定 B 结构，能否在 top-k 内找到标注的正确 A
- **解法质量**：LLM judge 对比 GPT-5 直接求解 baseline，打分比较

### 在线（产品化后）

- **top-3 采纳率**：用户点赞至少 1 个的比例
- **真实落地跟踪**：对可落地问题，跟踪用户是否实际采用某方案，效果如何（这是最硬的指标，但慢）

## 论文叙事升级

**原叙事**（当前 draft）：结构同构 embedding + 发现引擎 → 弱，"好玩的工具"
**新叙事**：

1. **框架**：创新 = 高语义距离 × 高结构同构 × **关键变形**
2. **V1**：语义距离可度量（SIBD 数据集 + embedding 模型）
3. **V2**：结构同构可自动发现（500 现象 → 24 tier-1 → N 个经验证的跨域类比）
4. **V3**：关键变形可预测（200 案例训练 + 历史回归测试 + 真新案例验证）
5. **组合**：三者合成"跨域求解器"，在 N 个真实问题上 beat GPT-5 直接求解 baseline

**目标投稿**：NeurIPS / ICML / Nature Machine Intelligence

这个叙事比 draft 硬一档：有数据集、有模型、有管道、有求解器、有 baseline 对比、有真实世界落地预测。

## 与验证管道的耦合

**验证管道必须先启动，原因**：
1. **训练数据**：V3 Phase 2 需要 tier-1 验证过的"真新"发现补充 10-20% 的现代案例
2. **论文压轴**：如果验证 Layer 3 产不出"强预测"发现，V3 论文的 Discovery 章节就是空的，叙事断裂
3. **V3 定位**：如果 24 条发现全是"灵感级"而非"求解级"，V3 方向可能要重新评估——做灵感工具不需要这么复杂的架构

**并行窗口**：
- Phase 1 不依赖验证结果（只要用户给种子就能启动）
- 建议：用户本周内给出 20-30 条种子案例 → Phase 1 立刻启动 → 与验证管道同步推进 → 两周后会师 Phase 2
