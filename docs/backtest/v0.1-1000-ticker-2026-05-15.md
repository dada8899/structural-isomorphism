# Backtest v0.1 — 1000-ticker walk-forward (2026-05-15)

> **Month-3 product positioning decision gate** per [W7-D Track A](../future/W7-D-product-value-roadmap-2026-05-13.md).
> Pre-registered hypothesis: companies classified as `near_critical` by the
> Phase Detector should show a different risk-adjusted forward return than the
> rest of a ~1000-ticker U.S. equity universe across a 5-year walk-forward.

## TL;DR — verdict

**Null result confirmed at scale.** Across 927 tickers and 59 monthly snapshots
(2020-12 to 2025-10), the LLM `near_critical` cohort achieves an annualised
Sharpe of **0.701** versus equal-weight benchmark **0.773**. The Sharpe
**lift is −0.072** (p = 0.569, two-sided paired t-test on monthly diffs).

At 6-month forward horizon, the lift is **−0.513** (p = 0.094, marginal). At
1-month forward horizon, the lift is **−0.072** (p = 0.569).

Across both LLM and price-based heuristic label sources, the consistent
ranking is: **`stable` cohorts > benchmark > `near_critical` cohorts**, but
no comparison clears α = 0.05.

**Positioning recommendation: pivot to structured-research-narrative.** The
Phase Detector label does *not* meet the W7-D Track A "Sharpe lift ≥ 0.5"
bar that would justify alpha-screener positioning at Pro $19/Team $99. Per
the W7-D pre-committed pivot plan, ship the product as a transparent
methodology + structured digest (Pro $9, B2B Index API), with the backtest
result published openly as a credibility moat.

## Methodology

### Universe (1001 tickers, 927 usable)

- **S&P 500** (500 tickers): from `companies_500.jsonl` Wave 1 input, each
  with LLM `critical_point_state` label (`deepseek-v4-flash` @ 2026-05-13).
- **Russell 1000 mid-cap supplement** (501 tickers): curated hand-sample of
  Russell-1000-but-not-SP500 names spanning Tech, Healthcare, Industrials,
  Financials, Consumer, Energy, Utilities, Materials, REITs, Media. Sourced
  from public iShares IWB holdings + Nasdaq.com Russell weights (April 2026).
- **Coverage**: 927/1001 = 92.6% tickers fetched. 74 dropped (delisted,
  yfinance dotted-ticker bugs like `BRK.B`, ETF symbols accidentally
  included). See `backtest/data/prices.meta.json` for failure list.
- **Survivorship bias warning**: dropped tickers are *current-name failures*;
  we make no attempt to include tickers that were delisted earlier in the
  window. This biases benchmark return upward (winners stay; losers
  disappear). Estimated bias = +1-2% per year on benchmark return — fully
  documented in "Honest interpretation" below.

### Price data

- **Source**: yfinance daily OHLCV (auto-adjusted), free, no API key.
- **Window**: 2020-01-01 to 2025-12-31 (5y, ~1500 daily bars per ticker).
- **Spans**: COVID crash (March 2020), zero-rate era (2020-2021), inflation
  + tightening cycle (2022-2023), AI rally (2023-2024), late-cycle (2025).
- **Cache**: per-ticker CSV at `backtest/data/prices/<ticker>.csv`. Reruns
  hit cache; ~1 min for incremental, ~5 min for cold fetch.

### Walk-forward

- **Snapshot grid**: month-end T from 2020-12-31 to 2025-10-31, step 1 month
  ⇒ **59 snapshots**.
- **Forward hold**: 1 month (headline). Sensitivity at 6-month horizon
  reported separately (48 snapshots, 2020-12 to 2024-11).
- **Cohort formation at T**:
  - LLM cohort: **static** — each SP500 ticker retains its 2026-05 label
    across the entire backtest window. This is a known label-leakage risk,
    explicitly carried over from v0.2 because per-snapshot re-labelling
    would cost $500-$1k in LLM calls (deferred to v0.3 if Track A pivot is
    walked back).
  - Heuristic cohort: **dynamic** — at each T, compute composite
    phase-stress score per ticker (90-day realised vol + 180-day max
    drawdown + 12-month return AR(1)). Top 10% decile = `near_critical`
    proxy; bottom 10% = `stable` proxy. This is a placebo / baseline against
    which the LLM label is measured.
- **Strategy**: equal-weighted long-only cohort return. We *did not* run
  long-short because (a) shorting carries borrow costs not modelled here,
  (b) the focal question is whether the LLM label is informative *at all*,
  not how to package it into a tradable strategy.
- **Benchmark**: equal-weighted mean monthly return across all 927 fetched
  tickers (proxy for an EW SP500+R1000 supplement basket).

### Statistical test

Paired t-test on monthly cohort-minus-benchmark return differences. n = 59
months. Two-sided null: mean diff = 0.

This is **one pre-registered test per cohort** — no multiple-testing
correction because no horizon/threshold/sector sweep was performed in this
report. Future sensitivity analyses must apply Benjamini-Hochberg.

## Per-snapshot dynamics

The full monthly time-series is in `backtest/results/v0.1-monthly-1000-universe-20260514T194239Z.csv`.

Key qualitative observations:

1. **2020-12 → 2021-12**: Risk-on regime. All cohorts up sharply (post-COVID
   recovery + zero-rate environment). `near_critical_heuristic` outperforms
   in this window — high-vol names rally hard. LLM `near_critical` keeps pace
   with benchmark.
2. **2022-01 → 2022-10**: Tightening shock. `near_critical_heuristic` craters
   (−56% max drawdown peak; high-vol names crash hardest). LLM `near_critical`
   drops in line with benchmark. `stable` cohorts hold up best.
3. **2023-01 → 2024-12**: AI rally. Benchmark recovers strongly. LLM
   `near_critical` and `other` both participate; no clear edge.
4. **2025-01 → 2025-10**: Late-cycle. Returns flatten; cohort dispersion
   shrinks.

Net: the LLM label gives essentially no timing alpha across regimes. The
heuristic high-vol cohort behaves exactly as financial theory predicts —
positive skew but Sharpe-destroying drawdowns. The price-stress placebo and
the LLM label are *not* the same thing, but neither produces a tradable
edge on monthly rebalance.

## Honest interpretation

### What this null tells us

1. **The Phase Detector label does not produce monthly-rebalance alpha** at
   the W7-D ≥0.5 Sharpe-lift threshold on a 927-ticker S&P 500 + Russell
   1000 mid-cap supplement universe over 2020-2025. The point estimate is
   slightly *negative* (−0.07 to −0.51 across horizons).
2. **Both LLM and heuristic labels suggest `stable` slightly outperforms
   `near_critical`**. The direction is wrong relative to a naive
   "phase-transition = upside vol opportunity" reading. It is right relative
   to a "low-vol-anomaly" reading — stable companies have less drawdown,
   compound more, and produce better Sharpe. The LLM label has captured
   *some* signal: companies it calls `far_from_critical` have +0.15 Sharpe
   lift over benchmark — but the t-stat (0.25) is far from significant.
3. **The heuristic placebo's poor performance** (Sharpe lift −0.28 at 1mo)
   is reassuring methodologically — if our results were driven by a hidden
   confound common to both label sources, we'd expect them to track each
   other. They don't: LLM `near_critical` lift is −0.07; heuristic is −0.28.
   The heuristic cohort is dominated by high-vol crash names (cf. 2022); the
   LLM cohort is more nuanced (it tagged 13% of SP500 as near-critical, not
   the top vol decile).

### Data-snooping discipline

- We pre-registered the 1-month forward-hold paired-t-test as the single
  primary statistical test (per W7-D Track A "Sharpe lift on near_critical
  cohort vs sector benchmark").
- 6-month horizon was reported as **one** sensitivity analysis, also
  pre-mentioned in W7-D.
- We did **not** sweep across all (horizon × threshold × sector × period
  start) combinations. Any future report that does so must apply BH
  correction (e.g. 4 horizons × 3 deciles = 12 tests ⇒ adjusted α = 0.0042).
- We did **not** select snapshots, tickers, or labels post-hoc to inflate
  the LLM cohort's performance. The Wave 1 StructTuple labels are the
  primary v0.1 artifact; we tested them as published.

### Limitations carried over from v0.2

These were known and stated in the v0.2 (500-ticker) report:

1. **Label leakage risk** — single-snapshot 2026-05 labels applied
   retroactively to 2020-2025 snapshots. Resolved only by per-snapshot
   re-classification (deferred for cost).
2. **Universe survivorship** — 74 dropped tickers may include earlier
   stress-points; this biases the benchmark upward.
3. **No transaction costs / borrow costs / shorting** modelled.
4. **Equal-weighted, no sector or vol weighting**.
5. **Benchmark = EW universe, not S&P 500 cap-weighted**. The latter would
   be ~+0.15 Sharpe higher (large-cap tilt) but is conceptually wrong for
   this universe (which includes mid-caps).

### What is *not* killed by this null

The Phase Detector remains valid as a **descriptive structural labelling
tool** even with zero alpha:

- The 5 dynamics families (preferential_attachment, motter_lai_cascade,
  scheffer_fold, etc.) still provide a defensible structural taxonomy.
- The per-company `structural_summary` + `evidence_anchors` still constitute
  audit-trail-grade research output.
- Customers may pay for **structured analysis at scale** even if no alpha is
  promised.

This is exactly the W7-D "Track B" structured-research-narrative
positioning: Substack-style + B2B Index API, Pro $9, no backtest claims.

## What we'd try if we had another month

Priority-ranked (cheapest → most expensive):

1. **Sector-stratified test** (cheap, 1 day). Run the t-test within each
   GICS sector. The signal may live in healthcare or tech and be diluted
   in financials/utilities. BH correction required (11 sectors).
2. **Dynamics-family stratified test** (cheap, 1 day). 9 families × 2 tests
   = 18 tests, BH adjusted. Was hypothesised in W7-D.
3. **Confidence-filtered cohort** (cheap, 0.5 day). Restrict
   `near_critical` to `confidence ≥ 0.8` labels only (54% of universe).
   Reduces cohort size but may sharpen signal.
4. **Per-snapshot LLM re-classification** (expensive, $500-$1k LLM cost,
   2-3 days incl. validation). This is the *single most likely fix* per
   the v0.2 report. If we pivot to research-narrative positioning, this is
   off the critical path; if a stakeholder argues "rerun with fresh
   labels", this is the line item.
5. **Russell 2000 universe** (medium cost). True small-cap is where phase
   transitions should bite. R2000 = 2000 names, fetch cost ~10 min, label
   cost ~$2-$5k if we LLM-classify the new 2000.
