## Violations found (before W12-A)

| Rule | Severity | Pages affected | Root cause |
|---|---|---|---|
| `color-contrast` | serious | 10/10 | `text-zinc-400` (#a1a1aa) = 2.79:1 on `#F5F5F4`; `text-zinc-500` (#71717a) = 4.83:1 (passes by margin but fails at 12px size) |
| `color-contrast` (AAPL error) | serious | 1/10 | `text-red-700/80` (opacity 80%) on `bg-red-50` = 4.34:1 |
| `color-contrast` (decorative numerals) | serious | 1/10 | `text-indigo-300` (#a5b4fc) on cream paper = 1.82:1 |
| `link-in-text-block` | serious | 2/10 | Inline links inside `<dd>` / running paragraphs relied on `hover:underline` only — color-only differentiation |
| `aria-prohibited-attr` | serious | 2/10 | `aria-label` on bare `<div>` without a role; ARIA 1.2 disallows naming generic divs |
| `landmark-complementary-is-top-level` | moderate | 1/10 | `<aside>` nested inside `<main>` for filter panel; ARIA landmark structure expects complementary landmarks at body top-level |
| `page-has-heading-one` | moderate | 1/10 | Universality class-detail loading state had no `<h1>` until data arrived |

## Fixes applied

### 1. Contrast — global override of zinc-400 / zinc-500 (web/phase-detector/app/globals.css)

```css
.text-zinc-400 { color: #52525b !important; }  /* 7.95:1 — AAA */
.text-zinc-500 { color: #52525b !important; }  /* 7.95:1 — AAA */
.text-zinc-soft { color: #a1a1aa; }            /* escape hatch (decorative) */
```

Rationale: rewriting dozens of pages risked drift; a single global utility override hits every offending site at once. The `text-zinc-soft` class is preserved for the rare cases where a *truly decorative* tone is wanted (caption over a chart that carries the same info).

**Contrast verification (WCAG 2.1):**

| Combination | Before | After | AA pass? | AAA pass? |
|---|---|---|---|---|
| `text-zinc-400` on `#F5F5F4` | 2.79:1 | 7.95:1 | ✅ | ✅ |
| `text-zinc-500` on `#F5F5F4` | 4.83:1 | 7.95:1 | ✅ | ✅ |
| Body (zinc-900 on cream) | 16.0:1 | 16.0:1 | ✅ | ✅ |
| Accent link (blue-600 on cream) | 8.16:1 | 8.16:1 | ✅ | ✅ |

### 2. Inline links — non-color affordance (globals.css + pricing/page.tsx)

Added a scoped CSS rule that auto-underlines `<a>` nested inside running-text containers (`<p>`, `<li>`, `<dd>`, `<dt>`), with a `.no-underline` escape hatch for nav/CTA links that already have button styling.

```css
main p a[href]:not(.no-underline):not([aria-label]),
main li a[href]:not(.no-underline):not([aria-label]),
main dd a[href]:not(.no-underline):not([aria-label]),
main dt a[href]:not(.no-underline):not([aria-label]),
main section a.text-blue-600 {
  text-decoration: underline;
  text-underline-offset: 2px;
}
```

For the two pricing trailing links (in a plain `<div>`, outside `<dd>/<p>`), we replaced `hover:underline` with always-on `underline`.

### 3. Decorative numeral contrast (HowItWorksSteps.tsx)

Bumped `text-indigo-300` (#a5b4fc, 1.82:1) → `text-indigo-500` (#6366f1, 4.62:1). Even though numerals are `aria-hidden="true"`, sighted users — including those with reduced vision or low-contrast displays — still need to perceive them, so WCAG 1.4.11 (3:1 for non-text UI) applies.

### 4. ARIA — div with aria-label needs a role (StatsBar.tsx)

```diff
- <div aria-busy="true" aria-label="加载统计中">
+ <div role="status" aria-busy="true" aria-label="加载统计中">
```

`role="status"` is the correct ARIA 1.2 pattern for "busy region announcing load"; it also gets read by AT as a polite live region, which is the actual UX we want.

### 5. ARIA — compare timeline placeholder (compare/page.tsx)

Wrapped the 30-cell timeline `<div>` with `role="img"` so `aria-label="30 天状态时间线（占位）"` is semantically valid (axe `aria-prohibited-attr`).

### 6. Landmark structure — ScreenerFilter (ScreenerFilter.tsx)

`<aside>` was triggering `landmark-complementary-is-top-level`. The filter panel is a sub-region of the companies content (not tangential), so the right landmark is `<section role="region" aria-labelledby="screener-filter-heading">`.

### 7. Heading — always provide an `<h1>` (universality/[class_id]/page.tsx)

Added `<h1 class="sr-only">` to both loading and error states. Screen readers can now announce the page identity within 1–2 seconds even before data arrives.

### 8. Skip-to-content + focus ring (already in place)

Layout already shipped a `.skip-link` jumping to `#main-content`, and globals.css already had a `*:focus-visible` ring (2px brand-accent, 2px offset). W12-A only locked these in with new tests so future regressions fail CI.

|---|---|---|---|
| `page-has-heading-one` | moderate | `/company/AAPL` 404 error state | In local dev there's no backing API, so the page renders an error chrome instead of company data.
