## Post body (single long-form, ~3500 chars)

A short note on a project I have been working on: testing whether systems from radically different scientific domains share the same underlying mathematical structure. The question is older than I am — universality classes in statistical physics show that a small set of equations describe phase transitions in materials, magnets, and fluids that look nothing alike. What I wanted to know is whether the idea, applied with the same statistical pipeline and no per-domain tuning, transfers to messier empirical domains: neural avalanches, financial contagion, DeFi liquidations, wildfires, citation cascades, GitHub starring dynamics.

I am treating it as a falsifiable question, not a positive thesis. The protocol commits to three discipline points:

(1) Pre-registered exponent bands. Every claimed universality class declares its expected power-law exponent and band before we touch new data. A fit outside the band is recorded as FAIL, not retroactively re-classified.

(2) Null controls. Four synthetic nulls (uniform, exponential, lognormal, shuffled) are run through the same pipeline. A framework that does not reject them is broken; ours does.

(3) Cross-judge LLM critic ensemble. A heterogeneous ensemble (Claude Sonnet, DeepSeek v4, Kimi K2.5, GLM-5) votes on candidate cross-domain pairs with explicit KEEP / REJECT / SPLIT / MERGE verdicts. No single model can wave a pair through.

The pipeline itself is one frozen Python module — v4/lib/soc_pipeline.py, 339 LOC, commit 7ee228c — implementing the Clauset–Shalizi–Newman 2009 discrete MLE with KS-optimal xmin selection, Hill-form alpha, block-bootstrap confidence intervals, and Vuong likelihood-ratio tests against lognormal and exponential alternatives. No per-system tuning anywhere downstream. The same function applied to thirteen empirical systems and four null controls produces the verdicts that go into the ledger.

Current state — and I want to be specific about the limitations:

- 13 of 13 empirical systems land in-band. That coverage is suspicious to me. The natural next test is pre-registering bands on fresh systems (Bitcoin Cash transaction sizes, FluNet ILI cascades, Flickr photo cascades, Bonabeau wasp dominance) before the data is fetched, to check whether the pipeline actually generalizes or whether the sample is selection-biased on the systems we already curated. That replication round is the active work.

- The "ensemble" is currently within-vendor (3x DeepSeek decodings at varied temperature). The architecturally diverse version (B4: Claude + GPT-5 + DeepSeek + Kimi + GLM-5) is partly blocked by region routing for some vendors from China-IP egress. We acknowledge this explicitly in v0.3 of the paper.

- One cross-domain collapse statistic (r_shape) in v0.3 was a combinatorial artifact. Fixed in v0.4 by replacing it with surrogate-null permutation tests. The point is not that the methodology is flawless — it is that the failures are recorded and the ledger is public.

The companion paper, "Adversarial Pre-Registration as Anti-p-Hacking Methodology: Four Negative and Partial Verdicts from a Shared Cross-Domain Validation Pipeline" (paper/anti-phacking-unified-2026-05-15.md), walks through four cases where the same pipeline returned FAIL / INCONCLUSIVE / PARTIAL / NULL verdicts: 2023 CVE high-severity disclosures, NYC FDNY 2023 fire dispatch unit sizes, r/wallstreetbets post cascades, and the commercial phase-detector S&P 500 walk-forward backtest. The argument is that the asymmetric distribution of outcomes (13 pass, 4 fail-or-partial out of 17 pre-registered) is more credible than 17 confirmations from a re-tunable pipeline would have been.
