## Body (no field; HN Show posts use the URL + comment)

I have been working on a falsifiable question: **do systems from radically different scientific domains share the same underlying mathematical structure?** Universality classes are a 50-year-old idea in statistical physics — a small set of equations describing phase transitions in materials, magnets, and lattices that look nothing alike. We wanted to see whether the same idea extends, with no per-domain tuning, to messier empirical domains: financial contagion, neural avalanches, DeFi liquidations, wildfires, GitHub starring, citation cascades.

The honest current state is: we have validated 13 empirical systems against pre-registered exponent bands using a single frozen 339-LOC Clauset MLE module. All 13 are in-band. That coverage is suspicious to us — eleven-out-of-eleven feels like the kind of result that should be falsifiable on a new system, not assumed. So the next phase is pre-registering exponent bands for fresh systems (Bitcoin Cash transactions, FluNet ILI, Flickr photo cascades) before fetching data, to see whether the pipeline actually generalizes or whether it is selection-biased on what we have already curated.

## What is in the repo

- **The pipeline.** `v4/lib/soc_pipeline.py` — one module, 339 LOC, frozen at commit `7ee228c`. Discrete Clauset power-law MLE with KS-optimal `xmin`, Hill-form `alpha`, block-bootstrap CIs, Vuong tests against lognormal and exponential. No per-system tuning anywhere downstream.
- **The dataset.** SIBD-63, on Zenodo at [10.5281/zenodo.19615170](https://doi.org/10.5281/zenodo.19615170) — 63 A-level cross-domain candidate pairs with shared equations, variable mappings, and full provenance. Curated by a multi-vendor LLM critic ensemble (Claude · DeepSeek · Kimi · GLM-5) with explicit KEEP / REJECT / SPLIT / MERGE verdicts.
- **A phase detector.** Research-preview consumer product at [phase.bytedance.city](https://phase.bytedance.city) — tags 100 public companies with their current dynamical phase (stable / accumulating / near-critical / reversed / recovering) against nine universality patterns. 500-ticker S&P 500 walk-forward backtest included. Research preview — not investment advice.

## Honest limitations

I want to be upfront, because this is the kind of project where it is easy to oversell:

- **The "ensemble" is currently within-vendor.** B3 is three DeepSeek decodings at varied temperature. That is multi-decoding, not cross-architecture. The architecturally-diverse version (B4: Claude Opus + GPT-5 + DeepSeek + Kimi + GLM-5) is blocked partly by OpenRouter region routing for Anthropic and Google from China IPs. We acknowledge this in v0.3 of the paper.
- **One statistic collapsed under scrutiny.** The `r_shape` cross-domain collapse number in v0.3 turned out to be a combinatorial artifact — fixed in v0.4 by replacing it with surrogate-null permutation tests.
- **Several `alpha` estimates sit close to the inverse-cubic boundary.** S&P 500 returns and GitHub star cascades both have `alpha` near 3.0 with CIs overlapping classical boundary regimes. The paper reports this; it is a real concern about how diagnostic the band test is in those cases.
- **Adjacent-domain transfer is unproven.** We validated 13 systems we already had reason to suspect lived in a power-law regime. The harder test — predicting whether a *new* system will pass the band — is the active work.

We treat the falsifiability discipline as the main deliverable. The companion paper (`paper/anti-phacking-unified-2026-05-15.md`) walks through four cases where the same pipeline returned **FAIL** / **INCONCLUSIVE** / **PARTIAL** / **NULL** verdicts (2023 CVE disclosures, NYC FDNY fire dispatch, r/wallstreetbets cascades, and a trading-signal commercial fork). We argue the asymmetric distribution of outcomes — 13 pass, 4 fail-or-partial out of 17 pre-registered — is more credible than 17 confirmations from a re-tunable pipeline would have been.

## What I would love HN for

1. **Reviewers willing to find holes.** The whole point of adversarial pre-registration is that the methodology is supposed to fail when applied to a system that should not pass. If you spot a system in our 13 where you think the band assignment is post-hoc, please open an issue.
2. **Domain experts to add datasets.** Fork, drop a dataset into `v4/validation/`, run `v4 validate <system>`, send a PR with the verdict. If your domain is *not* in the taxonomy, propose a pre-registered band.
3. **Bug reports.** Reproduction failures with environment + steps. We track these as P0.
