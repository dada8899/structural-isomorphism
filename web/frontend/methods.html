同一份代码,13 个独立系统

"共享模式"这件事只有"同一份代码、不调一个参数,跑遍多个完全不同的领域,都恢复出预测信号"时才算数。我们把这条要求做成了一个分层流程:从前几代检索积累的跨域现象,到 21 个候选共享模式,到 13 个独立数据源、全部跑同一份 Python 代码(查看代码 →)。

1)
现象收集 / 图构建
读取前几代检索的高质量结果(A 级 + B+ 发现),按统一打分阈值过滤,得到 180 节点 / 143 边的现象关联图。
2)
社区发现
用社区发现算法(Louvain)在关联图上聚簇,得到 23 个候选等价类,21 个跨 ≥ 2 领域。
3)
匹配到已知模式
把无监督浮现的等价类匹配到物理学已命名的共享模式(临界级联、强者愈强、电网级联、回不去效应、临界翻转、银行挤兑 …)。
4)
预测翻译
对已匹配的共享模式,把已知物理定律翻译为跨域可验证预测(幂律带、余震律、回路宽度比、临界放缓…)。
5)
实证验证(13 个独立领域)
同一份代码跑 13 个独立领域:地震 / 股市 / DeFi / 神经雪崩 / GitHub stars / 银行倒闭 / 野火 / 太阳耀斑 / 电网级联 / 维基百科 / 高速公路 / 湖泊翻转 + 4 个合成数据对照 + 7 系统共享形态对照。所有预测信号都落入预测带,对照组全部正确拒绝。

共享代码模块

所有领域共用同一份 Python 代码(查看代码 →)。一个函数对应一个分析操作:

  • 幂律拟合 — Clauset-Shalizi-Newman 2009 最大似然估计。
  • 置信区间 — Bootstrap 95% CI。
  • 对照假设比较 — Vuong 似然比,对比对数正态、指数分布。
  • 余震时间衰减 — Omori-Utsu 律(适用临界级联)。
  • 合成数据对照 — 匹配样本量的合成数据(高斯随机游走、指数、Poisson 事件、Poisson 余震)。
  • 贝叶斯模型选择 — 对数 binned BIC,幂律 + cutoff vs 对数正态。

每个领域调用同一个函数,数据接入是各领域专用的,但分析代码本身从不修改、不调参数、不加先验。这是"同一份代码"这件事的技术含义。

关键发现 — 共享错误模式: AI 审稿团推翻初始保留的 4 个候选(延迟微分债务、回不去效应 Preisach 作为单一类、无标度渗流类、尾部 copula 传染)共享同一种错误 — 把数学框架当作共享模式。这些都是数学基础设施,不是机制。三个独立审稿模型收敛于同一个判定本身就是非平凡的方法论验证。

局限: 同厂商 AI 审稿团探测的是同模型族内的判定漂移,不是跨架构分歧。跨厂商审稿(其他 LLM 家族)因地区访问限制推迟到后续版本。
技术细节

三个独立 AI 审稿模型:deepseek-v4-pro rigorous / deepseek-v4-flash rigorous / deepseek-v4-pro creative(T=0.6)。代码版本锁定在 commit 7ee228c,339 行 Python。同厂商 ensemble 探测 within-model confidence drift;跨厂商路径 Kimi / GLM-5 因 OpenRouter CN 区域 block + 备用 router 未验证 base URL 推迟到 B4+。

对照组与共享形态对照

为排除"代码把任何数据都拟合成幂律"的平凡解释,我们在 4 个合成数据上跑了同一份代码作为对照(包括 Gaussian 随机游走、指数变量、Poisson 事件、Poisson 余震叠加)。全部正确拒绝幂律假设(相对对数正态/指数分布的似然比 R ∈ [-45, -16],Poisson 余震拟合 R² ≈ 0.002)。

我们做了一道共享形态对照(让 7 个不同领域的曲线在归一化后比形状)。在有限尺寸标度假设下做归一化形态对照,达到 rshape = 1.11(优秀阈值 r < 2)。贝叶斯模型选择在 5/7 系统优先选幂律 + 指数 cutoff,对数正态在 0/7 系统胜出。

查看数学结构

有限尺寸标度形式: P(s) = s-α f(s/s*),其中 f 为归一化形状函数。形态崩塌评价指标 rshape 计算 7 个领域归一化曲线的形状偏差。
